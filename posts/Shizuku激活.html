Shizuku 激活教程

一、开启开发者模式（所有方式通用）

  1. 打开手机「设置」。
  2. 点击「关于手机」（或「关于本机」）。
  3. 连续快速点击「版本号」7 次，直到提示“你已处于开发者模式”。
  4. 返回设置首页，搜索「开发者选项」 或在「系统/更多设置」中找到进入。

二、关键权限配置

  • 在开发者选项中，开启「USB 调试」开关。
  • 重要 开启「无线调试」（Android 11+）及「禁止权限监控」 或 「USB 调试（安全设置）」（部分 ROM 叫法不同）。若未开启，ADB/无线调试可能报 Permission Denied。
  • Android 14+ 可在快速设置中添加“无线调试”瓦片，便于一键开启（长按通知栏 → 编辑快速设置 → 拖入开发者瓦片）。

三、方式一：无线调试激活（推荐 Android 11+ 非 Root 用户）

前提：Android 11 及以上，已连接 Wi-Fi。配对码可持久保存，但重启后需重新开启无线调试并启动 Shizuku。

  1. 打开 Shizuku 应用，点击“通过无线调试启动” → “配对”。
  2. 跳转到开发者选项 → 开启「无线调试」开关。
  3. 点击“无线调试”文字进入详情 → 选择「使用配对码配对设备」，记下 6 位配对码。
  4. 下拉通知栏，在 Shizuku 通知中输入配对码并确认（首次配对成功后可复用）。
  5. 返回 Shizuku 主界面，点击「启动」。
  6. 注意 点击后等待 5-10 秒，直到上方显示“Shizuku 正在运行”（可能显示 ADB 标识）。

小技巧 Android 14+ 可配合 Termux + 脚本实现半自动重启后激活（详见 GitHub 讨论）。

四、方式二：电脑 ADB 激活（适用于旧版 Android 或无线失败时）

  1. 用数据线连接手机到电脑，选择「传输文件」模式。
  2. 确保电脑已安装 ADB（平台工具），打开终端（Windows CMD / PowerShell / macOS Terminal）。
  3. 输入并执行命令（最新版路径，兼容旧版）：
adb shell sh /storage/emulated/0/Android/data/moe.shizuku.privileged.api/start.sh

或旧版路径：

adb shell sh /sdcard/Android/data/moe.shizuku.privileged.api/start.sh

终端显示 "exit with 0" 即成功。手机 Shizuku 界面稍等即可显示“正在运行”（ADB 标识）。

提示 首次成功后，可用 `adb tcpip 5555` 转为无线 ADB，后续无需线缆。

五、方式三：Root 激活（推荐已 Root 用户，最稳定）

前提：已 Root（如 Magisk、KernelSU、APatch），Root 管理器正常工作。

无需电脑、无配对、重启后通常一键恢复（配合后台权限接近开机自启）。

  1. 打开 Shizuku 应用。
  2. 首次弹出 Root 授权（如 Magisk 超级用户），授予权限并选择“始终允许”。
  3. 主界面直接点击 「启动」 或 「通过 root 启动」（自动识别 Root 模式）。
  4. 等待几秒，上方变为 “Shizuku 正在运行”（版本号后带 root 标识，UID 0 最高权限）。

优势 启动最快、权限最强，重启后点启动即可（无需配对/ADB）。

注意 实现接近开机自启：

  • 手机设置 → 应用 → Shizuku → 电池 → 无限制 / 关闭优化 / 允许后台。
  • Root 管理器中允许 Shizuku 后台、不杀进程。
  • 部分 ROM 关闭“深度休眠”/“自启动管理”限制。
  • 可选：早期 Sui 模块可真·开机自启（现多直接用 root + 后台权限足够）。

失败时：用 Root Checker 测 Root → 重启管理器 → 查看 Shizuku 日志。

⚠️ 激活状态为什么会失效？（通用 + Root 特别提醒）

Android 安全/省电机制导致服务易停，常见原因：

  • 重启手机：调试/Root 权限重置。非 Root 方式需重新开启无线调试/执行 ADB 命令。Root 方式通常只需重新点“启动”（后台未被杀前提下）。
  • 关闭开发者选项 / USB 或无线调试：立即失效（无线/ADB 方式明显）。
  • 后台被系统杀死：电池优化、激进省电（MIUI/ColorOS/One UI 等常见）。重要 设置 Shizuku 为无限制后台、关闭优化、锁定任务卡片、允许自启动。
  • Root 特别注意：
    • Root 管理器杀 Shizuku 进程或拒绝后台。
    • 未“始终允许” Root 权限，或 Root 临时禁用。
    • 激进杀后台 → 加常驻通知、避免冻结工具误杀。
    Root 优势：设置好后，重启点启动即恢复；部分用户用模块接近开机自启。
  • 其他： ROM Bug（Android 16+ 早期版有兼容，已在最新 Shizuku 修复）、版本过旧、模块冲突。建议用最新版（GitHub 下载，当前 \~13.6.x）。

快速自救：重启 Shizuku App → 授权 Root（若有）→ 点“启动” → 等待运行。若仍不行，查电池设置 / Root 授权 / 日志。
